## 1. Generics

Generics let you write one function or struct that works for any type, with the compiler generating a specialised version per type at compile time — zero runtime overhead:

```rust
// Without generics — duplicated for every type
fn largest_i32(list: &[i32]) -> i32 { *list.iter().max().unwrap() }
fn largest_f64(list: &[f64]) -> f64 { *list.iter().max_by(|a,b| a.partial_cmp(b).unwrap()).unwrap() }

// With generics — write once, works for any comparable type
fn largest<T: PartialOrd>(list: &[T]) -> &T {
    let mut biggest = &list[0];
    for item in list {
        if item > biggest { biggest = item; }
    }
    biggest
}

fn main() {
    println!("{}", largest(&[3, 1, 4, 1, 5, 9]));    // i32
    println!("{}", largest(&[3.1, 2.7, 4.8, 1.0]));  // f64
}
```

Generic structs work the same way:

```rust
struct Pair<T> { first: T, second: T }

impl<T: std::fmt::Display + PartialOrd> Pair<T> {
    fn larger(&self) -> &T {
        if self.first >= self.second { &self.first } else { &self.second }
    }
}
```

## 2. Traits

A trait defines a set of methods a type must implement — Rust's version of an interface:

```rust
trait Describable {
    fn describe(&self) -> String;                          // required

    fn short(&self) -> String {                            // default — can override
        let s = self.describe();
        s.chars().take(30).collect()
    }
}

struct Product { name: String, price: f64 }
struct User    { name: String, email: String }

impl Describable for Product {
    fn describe(&self) -> String { format!("{} — ${:.2}", self.name, self.price) }
}

impl Describable for User {
    fn describe(&self) -> String { format!("{} <{}>", self.name, self.email) }
}
```

**The orphan rule:** You can only implement a trait for a type if either the trait or the type belongs to your crate. You can't implement `Display` for `Vec<T>` — both are from `std`.

## 3. Trait Bounds

Use bounds to say "this generic must implement these traits":

```rust
// Inline bound
fn print_desc<T: Describable>(item: &T) { println!("{}", item.describe()); }

// where clause — cleaner with multiple bounds
fn print_sorted<T>(items: &mut Vec<T>)
where T: Describable + Ord
{
    items.sort();
    for item in items.iter() { println!("{}", item.describe()); }
}

// impl Trait — shorthand, same meaning as an inline bound
fn make_description(item: &impl Describable) -> String { item.describe() }

// impl Trait in return position — hides the concrete type
fn default_product() -> impl Describable {
    Product { name: "Widget".into(), price: 9.99 }
}
```

> **`impl Trait` return limit:** All code paths must return the *same* concrete type. If you need to return different types at runtime, use `Box<dyn Trait>` instead (see section 5).

## 4. Key Standard Traits

**`Debug` and `Display`**

```rust
use std::fmt;

#[derive(Debug)]  // free {:?} formatting
struct Point { x: f64, y: f64 }

impl fmt::Display for Point {  // you write {} yourself
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        write!(f, "({}, {})", self.x, self.y)
    }
}

let p = Point { x: 1.0, y: 2.0 };
println!("{p}");    // (1, 2)
println!("{p:?}");  // Point { x: 1.0, y: 2.0 }
```

**`From` / `Into`**

Implement `From<A> for B` and you get `Into<B> for A` automatically:

```rust
struct Metres(f64);

impl From<f64> for Metres {
    fn from(val: f64) -> Self { Metres(val) }
}

let m = Metres::from(3.5);     // explicit
let m: Metres = 3.5f64.into(); // implicit — same thing
```

**`Iterator`** — implement `next()` and get every adapter for free:

```rust
struct Counter { val: u32, max: u32 }

impl Iterator for Counter {
    type Item = u32;
    fn next(&mut self) -> Option<u32> {
        if self.val < self.max { self.val += 1; Some(self.val) } else { None }
    }
}

let sum: u32  = Counter { val: 0, max: 5 }.sum();                          // 15
let evens: Vec<u32> = Counter { val: 0, max: 10 }.filter(|x| x % 2 == 0).collect();
```

Common iterator adapters worth knowing:

```rust
let nums = vec![1, 2, 3, 4, 5];
let result: Vec<i32> = nums.iter()
    .filter(|&&x| x % 2 == 0)  // [2, 4]
    .map(|&x| x * x)            // [4, 16]
    .collect();

let product = nums.iter().fold(1, |acc, &x| acc * x); // 120
let pairs: Vec<_> = nums.iter().zip(["a","b","c","d","e"]).collect();
```

**`#[derive]`** auto-implements common traits when all fields support them:

```rust
#[derive(Debug, Clone, PartialEq, Eq, Hash, Default)]
struct UserId(u64);
```

| Derive | What you get |
|--------|-------------|
| `Debug` | `{:?}` formatting |
| `Clone` | `.clone()` method |
| `Copy` | copy-on-assign (stack types only) |
| `PartialEq` / `Eq` | `==` and `!=` |
| `Hash` | use as `HashMap` key |
| `Default` | `Type::default()` |
| `PartialOrd` / `Ord` | `<`, `>`, sorting |

## 5. Trait Objects — `Box<dyn Trait>`

When you need a collection that holds *different* types that all implement the same trait, use a trait object:

```rust
trait Shape { fn area(&self) -> f64; }

struct Circle  { radius: f64 }
struct Rect    { width: f64, height: f64 }

impl Shape for Circle { fn area(&self) -> f64 { std::f64::consts::PI * self.radius * self.radius } }
impl Shape for Rect   { fn area(&self) -> f64 { self.width * self.height } }

fn total_area(shapes: &[Box<dyn Shape>]) -> f64 {
    shapes.iter().map(|s| s.area()).sum()
}

fn main() {
    let shapes: Vec<Box<dyn Shape>> = vec![
        Box::new(Circle { radius: 3.0 }),
        Box::new(Rect   { width: 4.0, height: 5.0 }),
    ];
    println!("total: {:.2}", total_area(&shapes));
}
```

| | Generics `<T: Trait>` | Trait objects `dyn Trait` |
|-|-----------------------|--------------------------|
| Dispatch | Compile time (fast) | Runtime (vtable lookup) |
| Collection | One type per call site | Mix different types |
| Use when | Type known at compile time | Heterogeneous collections |

## 7. Summary

| Concept | Key Takeaway |
|---------|-------------|
| Generics | Write once; compiler specialises per type — zero runtime cost |
| Traits | Define a shared interface; types implement independently |
| Trait bounds | Constrain generics with `: Trait` or `where` |
| `impl Trait` | Shorthand bound; in return position, hides the concrete type |
| `Debug` / `Display` | Derive `Debug`; implement `Display` manually |
| `From` / `Into` | Implement `From`, get `Into` free |
| `Iterator` | Implement `next()`, get all adapters free |
| `#[derive]` | Auto-implements traits when all fields support them |
| `Box<dyn Trait>` | Dynamic dispatch for heterogeneous collections |
